## 概述

`/fp-1` 是一个 Claude Code Skill，用于在每次会话结束后自动复盘聊天记录，提取可复利的经验和踩过的坑，写入项目级沉淀文件和全局沉淀文件。

核心价值：把每次与 AI 协作的教训变成可检索、可复用的知识资产，避免重复犯错。

## 触发方式

- `/fp-1` — **基础模式**：仅复盘对话内容，提取可复利方法和踩坑记录
- `/fp-1 (all)` — **全面模式**：基础模式 + 任务完成度自检（验证刚才做的事是否真的做完了）

## 执行流程

### Step 0: 任务完成度检查（仅全面模式）

在复盘之前，先检查工作区里刚才做的事是否真的做完了。判断标准：

1. 用户明确要求完成某事（如"给所有文档加标签"）
2. 助手明确承诺完成（如"好的，我来处理"）
3. 实际结果验证：用 Read 或 Bash 确认文件确实存在且内容正确

如果发现未完成任务 → 立即继续执行，直到完成。

### Step 1: 定位沉淀文档

- 项目级沉淀：`project_XXX_项目名/沉淀_XXX_项目名.md`
- 全局沉淀：`workspace/01_全局沉淀.md`（不在 Obsidian vault 下创建）

### Step 2: 回顾聊天内容，提取两类信息

**A. 可复利方法**（通用性 + 可操作性 + 累积价值）
- 解决问题的新思路、新框架
- 被验证有效的操作流程
- 工具的新的高效用法

**B. 踩坑记录**（教训价值 + 具体性 + 因果关系）
- 操作失误及其纠正过程
- 预期与实际不符的情况
- 花了额外时间才搞定的事

### Step 3: 分层写入

1. 所有提取内容先写入项目沉淀
2. 筛选出具有公共性/通用性的内容追加到全局沉淀
3. 判断标准：不依赖特定项目业务逻辑、适用于多个项目、工具/系统/流程层面的共性教训

### Step 4~6: 格式化 → 写入 → 报告

## 重要规则

1. **宁缺毋滥** — 只有真正有复利价值的方法和有教训意义的坑才记录
2. **不重复** — 已有内容不重复添加
3. **简洁具体** — 每条 1-2 句话，说清核心要点
4. **项目沉淀为主，全局沉淀为辅**

## 配套 Skill

- `/fp-2` — 沟通问题诊断，审视用户自身的沟通质量（与 fp-1 互补，fp-1 复盘技术经验，fp-2 复盘沟通方式）

## 沉淀文件结构

全局沉淀 `01_全局沉淀.md` 包含三个板块：

| 板块 | 用途 |
|------|------|
| 踩坑记录表 | 日期 / 项目 / 问题 / 根因 / 解决 / 教训 |
| 可复用方法表 | 方法名 / 适用场景 / 简述 / 出处 |
| 决策记录表 | 日期 / 项目 / 决策 / 选择 / 理由 / 结果 |
| 复盘记录表 | 日期 / 复盘轮次 / 摘要 |

## 使用示例

```
# 会话结束后，运行复盘
/fp-1

# 全面复盘 + 任务验证
/fp-1 (all)
```

## 文件位置

本 Skill 安装在 `~/.claude/skills/fp-1/SKILL.md`，每次对话中通过 `/fp-1` 斜杠命令触发。
