## 概述

`/frontmatter-1` 是一个 Claude Code Skill，为 Markdown 笔记自动添加或修正 YAML frontmatter。支持单文件模式和批量处理模式。

## 触发方式

- `/frontmatter-1` — 处理当前笔记
- `/frontmatter-1 path/to/note.md` — 指定文件
- **批量处理**：用户要求处理多个文件时自动启用 Workflow 并行模式

## 校验规则

已有 frontmatter 也会检查格式，格式不对则整体替换（不修补单个字段）：

| 字段 | 要求 | 错误示例 |
|------|------|----------|
| `abstract` | 必须存在，字符串，≤40字 | 缺失、空值 |
| `author` | 必须存在，固定 `cc` | 缺失、值不是 `cc` |
| `created` | 必须存在，`YYYY-MM-DD` | 格式不对 |
| `tags` | 必须存在，YAML 列表 | 行内数组 `[tag1]` |
| `url` | 可选，必须合法 http/https | 存在但格式不对 |

## 自动生成逻辑

### Abstract
读完全文，用一句简洁中文总结核心内容。

### Tags
- 基于内容智能推荐 2-4 个标签
- 优先使用 vault 已有标签体系（AI工具、ClaudeCode、整理、方法论等）
- **必须额外添加 PARA 分类标签**（见下方规则）

### PARA 分类判断

**第一步：从文件夹路径初步推断**

| 象限 | 路径关键词 | 标签 |
|------|-----------|------|
| 项目 (P) | 任务、在做、项目、产出 | `PARA/项目` |
| 领域 (A) | 复盘、认知、每周、身份 | `PARA/领域` |
| 资源 (R) | ai、工具、指南、沉淀、知识库 | `PARA/资源` |
| 归档 (Ar) | 归档、archive、历史、旧 | `PARA/归档` |

**第二步：结合文档内容修正**

- 有明确目标/截止日期/里程碑 → P（项目）
- 持续维护的责任区/长期积累 → A（领域）
- 参考资料/方法论/工具说明 → R（资源）
- 已完成/不再活跃 → Ar（归档）

**冲突时以内容为准**。每个文档优先只加一个 PARA 标签。

## 写入策略

- **没有 frontmatter**：在文件开头第一行前插入
- **已有 frontmatter 但格式错误**：替换整个 frontmatter 块（从第一个 `---` 到闭合的 `---`）
- **格式正确**：跳过，报告"已有 frontmatter，格式正确"

## 批量处理模式

当用户要求处理多个文件时，使用 `Workflow` 工具并行处理：

1. Glob 发现目标文件夹下所有 `.md` 文件
2. 启动 Workflow，每个文件由独立 agent 处理
3. 处理完后汇总报告

## 重要规则

1. **已有也检查** — 格式正确则跳过，错误则整体替换
2. **信任写入，不询问** — 生成后直接写入
3. **标签用中文** — 优先匹配 vault 现有标签体系
4. **abstract 要准确** — 基于全文内容，不能空洞
5. **不修改正文** — 只在文件顶部操作 frontmatter 块
