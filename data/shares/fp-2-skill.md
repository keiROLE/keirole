## 概述

`/fp-2` 是一个 Claude Code Skill，分析用户在对话中的沟通方式，指出**具体问题**和**改进建议**，帮助用户提升与 AI 协作的效率。

与 `/fp-1`（复盘技术经验）互补：fp-1 关注"项目层面的坑和方法"，fp-2 关注"用户自身的沟通质量"。

## 触发方式

- `/fp-2` — **基础模式**：指出本轮对话中的沟通问题 + 改进建议
- `/fp-2 (all)` — **全面模式**：基础模式 + 读取记仇本 + 结合历史共性问题诊断

## 问题分类

### A. 信息缺失
- 缺少关键约束（平台、版本、环境）
- 缺少上下文（没说明之前做过什么、当前状态）
- 缺少目标描述（没说清楚想要什么结果）
- 省略必要细节（文件路径、错误信息、日志）

### B. 指令模糊
- 用词含糊（"优化一下""弄好看点"，没说清具体怎么优化/好看）
- 多个意图混在一起，没有分层
- 用主观形容词代替客观描述（"大一点""快一点"）

### C. 表达冗余
- 大量铺垫和背景，结论/需求藏在最后
- 同一件事反复说，每次说法不同
- 情绪性表达多于事实性描述

### D. 假设错位
- 假设 AI 知道某个上下文但实际不知道
- 假设某个工具/方法能做到但实际做不到
- 假设文件位置/文件名/接口行为但实际不同

### E. 缺乏结构化
- 一次问太多不相关的事
- 没有区分优先级
- 没有给出示例或参照物

## 评估与输出

每个问题标注严重程度：
- **严重** — 导致反复返工、长时间卡住、完全走错方向
- **中等** — 增加了沟通成本、需要追问确认
- **轻微** — 有小问题但不影响最终结果

输出格式：
```
[严重等级] [问题类别]
- 问题：...
- 影响：...
- 建议：...
```

## 全面模式的记仇本机制

`/fp-2 (all)` 会读取 `workspace/04_ai记仇本.md`，记录用户反复犯的沟通错误：

| 日期 | 问题类别 | 典型表现 | 严重等级 |
|------|----------|----------|----------|

- **已有记录的问题**：本轮又犯 → 加重标记，追加新记录
- **新问题**：首次出现，严重/中等 → 追加到记仇本
- **已消失的问题**：记仇本中有但本轮没出现 → 标注"最近未再出现"

## 使用示例

```
# 会话结束后，诊断沟通质量
/fp-2

# 全面诊断 + 对照历史记录
/fp-2 (all)
```
